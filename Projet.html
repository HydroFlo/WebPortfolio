Étudiant en BUT info, parcours réalisation d'applications

Projets

  • Othello

    • Recréation complète du jeu Othello en Java dans le cadre d'un mini-projet Agile en groupe de 6, en appliquant les principes de la méthodologie Agile (décomposition des tâches, estimation de leur poids respectif et répartition en sprints de 1h30). Le jeu fonctionne dans le terminal et permet de jouer une partie d'Othello soit contre un autre joueur en local, soit contre un ordinateur.

  • Application de calcul d'itinéraire

    • Cette application permet, à partir d'un fichier CSV contenant des routes entre deux points (par exemple, des villes), le mode de transport emprunté (train, avion, bus) et des données pouvant être intéressantes pour choisir un trajet plutôt qu'un autre (coût en CO2, temps, prix), de générer un graphe permettant de déterminer le chemin le plus court pour aller d'un départ à une destination, tout en favorisant les critères sélectionnés par l'utilisateur.

  • Labyrinthe Perdu

    • Jeu réalisé en Java (à l'aide d'une bibliothèque fournie par le BUT). Il s'agit d'un jeu ludopédagogique qui se lance dans un terminal. Le personnage peut se déplacer à travers un labyrinthe généré aléatoirement, où il pourra rencontrer des monstres qui lui poseront des questions. Si vous répondez correctement, vous augmentez votre score ; sinon, vous perdez de la vie. Le jeu se termine dès que votre vie atteint 0 ou si vous parvenez à vaincre le boss au centre du labyrinthe.

  • Application implémentant l'algorithme K-NN

    • Cette application permet, à partir de l'algorithme des plus courts chemins (K-NN), de déterminer l'appartenance d'une donnée parmi un jeu de données déjà connu. Par exemple, imaginons que l'on possède un jeu de données représentant des Iris ayant différentes caractéristiques et dont on connaît l'espèce de chaque plante. D'après l'algorithme K-NN, on peut trouver l'espèce d'un Iris inconnu à partir de ses caractéristiques et de sa proximité avec les autres espèces.
    Cette application a été réalisée en Java pour toute l'implémentation et via JavaFX pour l'interface. Nous étions à cinq sur ce projet et avons utilisé Git pour cela.
